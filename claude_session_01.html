12 — Error reporting overhaul (log + Errors sheet + Status)

Assistant

Old behavior: button function raised → xlwings/VBA showed Excel's truncated MessageBox; "Press Ctrl+C to copy" didn't work.

New behavior: button_* functions catch all exceptions and surface them in three places:

  1. stocks_errors.log next to the workbook — full traceback, append-only.
  2. An Errors sheet inside the workbook (created on first failure) — one row per failure.
  3. The Main!Status cell — concise summary + hint.

No re-raise → no truncated popup. Each of the three writes is independently try/except-guarded so the handler itself can never crash.

15 — xlwings tests killing user's Excel — skip guard

User

Screenshot: "Those new tests kill other Excel sessions". Fix this

Assistant

Root cause: on some Excel/COM combos, xw.App() can bind to an existing user instance, and app.quit() on teardown then closes the user's workbooks (destroying unsaved work).

Fix: at the top of the test fixture, check xw.apps — if non-empty, pytest.skip the whole module with a clear message: "Excel is already running. Close it and re-run if you want this layer covered." Verified: 9 tests skip cleanly with Excel open. f2ec560

Saved the rule to memory: any test that creates an xlwings App MUST skip when Excel is running.
