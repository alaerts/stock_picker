3 — Cell-based STOP + busy-flag + portfolio validation

User wanted a STOP button. Technical constraint: xlwings RunPython is synchronous, so a button macro queues — only Form Controls update LinkedCells instantly during a VBA block.

User

Implement the "In-Excel cell-based stop". Trigger is another button, not a checkbox. While a macros is running, disable the buttons except the Stop button. Trigger an error if you cannot find one of my owned stock.

Assistant

I explained the constraint: xlwings RunPython blocks the VBA thread synchronously. A button click during that block would queue — Excel can't dispatch it until Python returns. Only Form Controls (checkboxes) update their LinkedCell instantly via Excel's internal cell engine, bypassing the VBA thread. Offered three options:

  1. STOP checkbox writing TRUE to Main!B13 (works, but not a button)
  2. Async/AppDoEvents in VBA (fragile, breaks under .info loops)
  3. Drop the stop feature
User

okay to go with number one

Assistant

Implemented (fd0203c):

  • Cell-based STOP: MAIN_CELLS["StopRequested"]="B13", polled every 25 tickers inside fetch_all_info. On True, breaks the loop and returns the partial dict. Caller detects via len(out) < len(tickers).
  • Busy-flag guard: MAIN_CELLS["JobRunning"]="B14". VBA wrapper sets it TRUE at job start, FALSE on cleanup. GuardJobStart() refuses to launch a second job — prevents accidental double-clicks.
  • Owned-stock validation: new _check_portfolio_symbols_resolve() raises RuntimeError if any Main!Portfolio entry doesn't match a constituent (with or without exchange suffix). Catches typos like "KBC" when the workbook stores "KBC.BR".

Wired into all 4 paths (CLI rebuild, CLI get_quotes, button rebuild, button get_quotes). 9 new tests, all green.

4 — Speedups A+C+F (parallel .info + freshness + SQLite)

User asked for the three speedup options I'd proposed earlier. Implemented all three in one commit (42856b5).

Assistant

Speedup A: fetch_all_info rewritten to use ThreadPoolExecutor(max_workers=12). The previous sequential time.sleep(0.25) per ticker is gone; concurrency cap is the new rate-limiter. Progress + should_stop callbacks preserved via as_completed. Pending futures cancelled on STOP.

Cache file stocks_info_cache.sqlite sits next to the workbook (gitignored). fetch_all_info_with_cache returns (info_map, cache_hits) so status text can surface the hit count.

8 — Batch 1 reverted: yfinance requires curl_cffi

User flagged the CSX5.AS adoption error and asked me to add a test. Investigating it surfaced that Batch 1 had been silently destroying yfinance.

User

For CSX5.AS, you returned the error "Symbol not found on Yahoo Finance (CSX5.AS)" yet there is https://finance.yahoo.com/quote/CSX5.AS/.

Assistant

Direct test: yf.Ticker('CSX5.AS').info returns 82 fields including regularMarketPrice=228.75, longName="iShares VII PLC...". Symbol obviously exists. Then I tried with the shared session from Batch 1:

no session:   {'Name': 'iShares VII PLC...', 'currency': 'EUR'}
with session: None

The shared session was breaking yfinance. Verified with yf.download:

YFDataException: Yahoo API requires curl_cffi session
not <class 'requests.sessions.Session'>.
Solution: stop setting session, let YF handle.

yfinance 0.2.x rejects plain requests.Session — it requires curl_cffi for TLS fingerprinting. The maintainers' recommendation is the OPPOSITE of what I did. Reverted all session passing (14118e8). Kept the bits that didn't depend on session: retry-on-empty, parallel fetch_close_prices, parallel fetch_all_watchlists.

Lesson: When a library has explicit error messages telling you what NOT to do, listen. The 401s we observed were already being retried by yfinance internally (visible in ERROR logs but invisible in results — "982 ok, 0 failed"). The "structural fix" was solving a non-problem AND introducing a real one.

11 — Three more bugs: AutoFilter, I1 bold, comma style

User inspected the workbook and reported three problems in one message.

User

Turn on filtering in all 4 result sheets. In the monthly winner sheet, cell I1 formatting is wrong. Use comma style formatting on all currencies.

Assistant

Inspected workbook via openpyxl. All three bugs confirmed (5c90809):

  1. AutoFilter missing on all 4 result sheets — even Market! Root cause: Excel strips openpyxl-set <autoFilter> elements on manual save. rebuild_inventory was the only path that set Market's filter, so once Excel stripped it the state persisted forever. Fix: get_quotes (CLI + xlwings) now re-applies Market AutoFilter at the end of every run. All 4 sheets self-heal on every CLI or button run.
  2. I1 bold=False — only I1 was unstyled. Root cause: _ensure_ranking_sheet_openpyxl only applied bold + fill when is_new=True. When v03 added Owned? to RANKING_HEADERS, the new I1 got its value rewritten but no styling. Same family of bug affects _layout_market_sheet (v04 Industry would have hit it on Market) and _ensure_currencies_sheet_openpyxl. Fix: re-apply Font.Bold + PatternFill whenever a header VALUE is rewritten. Preserves user cosmetics on already-correct cells; styles new schema-bump cells.
  3. Comma style on currencies — was "#,##0.0000" (plain comma). Switched to accounting-style '_-* #,##0.0000_-;-* #,##0.0000_-;_-* "-"????_-;_-@_-' matching Market's EUR price columns. Kept at 4 decimals (2dp would round 1.1735 → 1.17 and hide intra-week FX moves).
